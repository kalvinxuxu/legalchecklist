## 安全机制分析

### 租户隔离实现层次

```
┌─────────────────────────────────────────────────────────────┐
│                       API 层 (Endpoints)                     │
├─────────────────────────────────────────────────────────────┤
│  1. JWT Token 身份验证 (get_current_user)                    │
│  2. 租户隔离中间件 (verify_*_access)                         │
│  3. 查询条件过滤 (WHERE tenant_id = ?)                       │
├─────────────────────────────────────────────────────────────┤
│                      数据库层 (SQLAlchemy)                   │
└─────────────────────────────────────────────────────────────┘
```

### 关键安全代码

**中间件**: `backend/app/middleware/tenant_isolation.py`

```python
async def verify_workspace_access(
    workspace_id: str,
    current_user: UserModel = Depends(get_current_user),
    db: AsyncSession = Depends(get_db)
) -> Workspace:
    """验证用户是否有权访问指定工作区"""
    result = await db.execute(
        select(Workspace).where(
            Workspace.id == workspace_id,
            Workspace.tenant_id == current_user.tenant_id  # 租户隔离条件
        )
    )
    workspace = result.scalar_one_or_none()
    if not workspace:
        raise HTTPException(404, "工作区不存在或无权访问")
    return workspace
```

**关键点**:
- 所有查询都包含 `tenant_id == current_user.tenant_id` 条件
- 防止 ID 遍历攻击（即使知道其他租户的 ID 也无法访问）
- 统一返回 404 而非 403，避免泄露资源存在性
